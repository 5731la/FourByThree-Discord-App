# FourByThree Discord App

A Discord Activity that lets you play Hank Green's daily [4×3 puzzle](https://www.hankgreen.com/fourbythree) and [Smush](https://www.hankgreen.com/smush/) directly inside a Discord channel, and effortlessly share your results back to the chat.

The game itself is served from hankgreen.com — this project is a Discord Activity shell and Go Reverse Proxy that injects the Discord Embedded App SDK and handles native integrations like Slash Commands, OAuth2 Authentication, and Interaction Webhooks.

Before getting to deep into the code, understand that this was mostly generated by LLMs. Code quality is probably low, but also this is fragile by design, so this is the standard I am writing this to.

## Features
- **Two Games**: Supports both 4×3 (`/play4x3`) and Smush (`/playsmush`).
- **Direct Proxying**: Serves the live game directly while injecting the Discord SDK.
- **Slash Commands**: Start 4×3 with `/play4x3` (supports custom puzzle links). Start Smush with `/playsmush`.
- **Auto-Updating Results**: When you finish the game, the proxy intercepts the result image and automatically posts it to the channel, pinging you and displaying your score as alt-text!
- **OAuth2 Token Exchange**: Seamlessly identifies the player without requiring explicit Bot channel permissions.

## Complete Deployment Guide

### 1. Discord Developer Portal Setup
Before running the code, configure your application in the [Discord Developer Portal](https://discord.com/developers/applications).

1. Create a new Application.
2. Go to **General Information**:
   - Note your **Application ID** (Client ID) and **Public Key**.
   - Set the **Interactions Endpoint URL** to your proxy: `https://your-domain.com/discord/interactions`
3. Go to **OAuth2 -> General**:
   - Note your **Client Secret**.
   - Under **Redirects**, add EXACTLY your domain (e.g. `https://your-domain.com`). This is absolutely required for token exchange to work.
4. Go to **Installation**:
   - Leave settings at defaults (or configure for User Install/Guild Install as desired).
5. Go to **Activities**:
   - Enable the Activity feature.
6. Under **URL Mappings**, add:
   | Prefix | Target |
   |---|---|
   | `/` | `your-domain.com` (your tunnel/VPS URL — no `https://`) |

### 2. Configure Environment Variables
Create a `.env` file based on `.env.example`:

```bash
cp .env.example .env
```

Edit `.env` and fill in:
- `DISCORD_CLIENT_ID`: Your Application ID.
- `DISCORD_CLIENT_SECRET`: Your Client Secret (OAuth2).
- `DISCORD_PUBLIC_KEY`: Your Public Key (for verifying Webhook interactions).
- `DISCORD_REDIRECT_URI`: The exact Redirect URI you registered in step 1.3 (e.g., `https://your-domain.com`).
- `PORT`: (Optional) Default is `4343`.

### 3. Fetch the Discord SDK
The JavaScript SDK is gitignored and must be vendored before building:

```bash
npm pack @discord/embedded-app-sdk@1
mkdir -p client/static
tar -xzf discord-embedded-app-sdk-*.tgz --strip-components=2 -C client/static/ package/output
rm discord-embedded-app-sdk-*.tgz
```

### 4. Register Slash Commands (One-time Setup)
You must tell Discord that your app responds to `/play4x3` and `/playsmush`. Run both commands from any terminal:

**4×3:**
```bash
source .env
curl -X POST \
  -H "Authorization: Bot $DISCORD_BOT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "play4x3",
    "description": "Start a game of 4x3 in this channel!",
    "type": 1,
    "options": [
      {
        "name": "link",
        "description": "Optional custom puzzle link (e.g. https://www.hankgreen.com/fourbythree#b=...)",
        "type": 3,
        "required": false
      }
    ]
  }' \
  "https://discord.com/api/v10/applications/$DISCORD_CLIENT_ID/commands"
```

**Smush:**
```bash
source .env
curl -X POST \
  -H "Authorization: Bot $DISCORD_BOT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "playsmush",
    "description": "Start a game of Smush in this channel!",
    "type": 1
  }' \
  "https://discord.com/api/v10/applications/$DISCORD_CLIENT_ID/commands"
```
*(Note: You only need a Bot Token once to register commands. Add `export DISCORD_BOT_TOKEN=your_token_here` to your `.env` for this step — the running proxy itself does NOT require it!)*

### 5. Build and Run
```bash
source .env
go build -o fourbythree-discord-app .
./fourbythree-discord-app
```

### 6. Production (NGINX)

`hankgreen.com` uses Next.js streaming (chunked transfer encoding). To ensure the proxy doesn't buffer the response (which would cause the client to wait 60s+ for the stream to close), use this NGINX config:

```nginx
server {
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:4343;
        proxy_http_version 1.1;           # required for chunked encoding
        proxy_set_header Connection "";   # enable keep-alive to upstream
        proxy_set_header Host $host;
        proxy_buffering off;              # stream chunks immediately
    }
}
```
